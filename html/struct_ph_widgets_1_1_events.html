Emitted by the Photon Manager on behalf of a region being moved, resized, or removed from the event space.

info 

All regions must always be transparent to 'Events::info' events. They are emitted by applications or service providers to disseminate information or respond to requests.

key 

Emitted when a key state changes (for example, the user presses or releases a key).

Emitted when the user moves the pointing device while pressing a button.

Emitted when the user moves the pointing device without pressing a button.

raw 

These are raw, unfocused events that the Photon server handles.

service 

These events may be emitted by applications requesting services or providing information to services, other applications that provide some kind of service in a Photon system.

system 

Are emitted when Photon or a service wants to inform applications of changes in the system.

timer 

Emitted by an application directly to the Device region to request a reciprocal event after a specific amount of time has elapsed (arm a timer).

user 

A custom event type that your application can use to interact with other applications.

wm 

Both the Window Manager and applications can emit this event. The Window Manager emits this event when an application has asked to be notified. An application can emit this event to communicate to the Window Manager regarding windows.
